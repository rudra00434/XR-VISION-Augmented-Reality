# AERO-G1 · Hybrid XR Vision System (v4.0.0)

![AERO-G1 Vision System](https://img.shields.io/badge/XR-VISION-00dcff?style=for-the-badge&logo=webxr)
![A-Frame](https://img.shields.io/badge/A--Frame-1.7.0-ef2d5e?style=for-the-badge&logo=a-frame)
![Vite](https://img.shields.io/badge/Vite-6.2.0-646cff?style=for-the-badge&logo=vite)

A high-performance, premium WebXR prototype for interacting with 3D product models in both **Augmented Reality (AR)** and **Virtual Reality (VR)**. Optimized for mobile responsiveness and low-latency interaction.

---

## 🚀 Key Features

- **🥽 Dual-Mode Launch**: Choose between `Live AR Vision` (using the camera and hit-test) or `Virtual Gallery` (immersive VR environment).
- **🏎️ Fluid Architecture**: Responsive typography and UI scaling that feels like a native app on any device.
- **🛰️ Advanced HUD**: Professional telemetry overlay showing active mode, sensor status, and interactive product details.
- **⚡ Performance Optimized**: Minimal dependency stack using A-Frame 1.7.0 and Vite for instant load times.
- **📱 Hybrid Fallback**: Smart detection of WebXR capabilities with automated fallback to device-camera streams if native AR is unavailable.

---

## 🛠️ Tech Stack

- **Core**: [A-Frame 1.7.0](https://aframe.io/) (WebXR Framework)
- **Logic**: [Vanilla ES6+](https://developer.mozilla.org/en-US/docs/Web/JavaScript)
- **Styling**: [CSS Modern Hooks](https://developer.mozilla.org/en-US/docs/Web/CSS) (Fluid Typography & Glassmorphism)
- **Build**: [Vite](https://vitejs.dev/)

---

## 🏁 Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) (v16+)
- [npm](https://www.npmjs.com/)

### Installation

```bash
# Clone the repository
git clone https://github.com/rudra00434/XR-VISION-Augmented-Reality.git

# Enter the directory
cd XR-VISION-Augmented-Reality

# Install dependencies
npm install
```

### Development

```bash
# Start the local server
npm run dev
```
